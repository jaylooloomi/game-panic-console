# Panic Console: 20s Overload — 專案定案規格 (SCOPE)

> 本文件記錄「**已確定**」與「**待定**」的範圍劃分，作為後續設計與開發的基準。
> 最後更新：2026-06-14
>
> **層級說明**
> - 🔒 **確定 (Locked)**：不可隨意更動，更動需明確重新拍板。
> - 🎚️ **提案預設值 (Default, 可調)**：已有建議數字，但屬平衡性參數，做原型時可微調。
> - ❓ **待定 (Open)**：尚未決定，後續討論後由設計規劃。

## 🔒 1. 已確定（鐵則）

| 項目 | 內容 |
|------|------|
| **遊戲名稱** | `Panic Console: 20s Overload`（中文：心跳控制台：20秒超載） |
| **上架平台** | Steam |
| **開發引擎** | **Unity (C#)** — 2026-06-14 定案；Steam/連線整合最成熟、AI 協助開發最可靠 |
| **直播核心** | 直播主可連線對戰；觀眾能參與遊玩（以指令／投票干擾賽局） |

## 🔒 2. 核心遊戲特性（已確定 — 遊戲靈魂，不可動）

| 特性 | 說明 |
|------|------|
| **20 秒強制切換** | 螢幕每 20 秒強制切換到另一個小遊戲（名稱裡的 USP，不可動） |
| **多工地獄** | 同時運行多個小遊戲；後台持續運作，失誤一樣扣血 |
| **像素風格** | 視覺定調：太空艙控制台 + 像素小遊戲 |
| **卡牌干擾／驚嚇系統** | 玩家互相施放干擾卡（女鬼閃現、按鍵反轉、螢幕噴墨等）整對手 |
| **觀眾互動干擾** | 觀眾透過聊天室指令／投票影響賽局（直播核心的具體玩法） |
| **經典小遊戲重製合集** | 以大眾熟悉的經典玩法（俄羅斯方塊、貪食蛇等）降低學習門檻 |

## 🎚️ 3. 提案預設值（可調，非鐵則）

> 來自規格書 Ver 1.2，先當預設值，做原型時依手感微調。

| 參數 | 預設值 |
|------|--------|
| 共享生命值 | 5 點（失誤扣 1 點） |
| 切換間隔遞減曲線 | 20 → 15 → 10 秒 |
| 同時並行小遊戲數 | 3 個 |
| 能量／抽卡 | 集滿 100% 抽卡；卡牌槽上限 3 張 |
| 卡牌持續時間 | 女鬼 0.8 秒、噴墨／反轉 5 秒…（詳見規格書 2.2） |
| 英雄角色數 | 4 種（駭客／工程師／偶像／武士） |
| 小遊戲數量 | 15+ |

## ❓ 4. 待定（後續討論後規劃設計）

- **串流平台範圍**：只做 Twitch？或 Twitch + YouTube 都要？
- **小遊戲最終清單**：首發做幾個？哪幾個？是否真要 15+？
- **英雄角色最終清單**：4 個夠嗎？技能如何定？
- **Mobile 跨平台**：規格書僅「預留接口」，首發是否納入？
- **單人 vs 多人範圍**：是否有單人生存模式？1v1 之外是否擴充？
- **商業模式／定價**：買斷？價格帶？
- **玩家自製小遊戲（JS API + Steam 創意工坊）**：首發納入或後續版本？
